# AppForge デモ — TODO アプリ自動生成(Playwright)

GUI 操作だけで **3 ページ + 画面遷移(ナビゲーション)**のある TODO アプリを組み立て、
プレビューで遷移を確認し、最後に React を実ビルドするまでを Playwright で自動再生します。
画面右上にシナリオのチェックリストを表示し、次に操作するパーツ/ボタンをハイライト
(黄色いグロー)しながら進みます。何度でも繰り返し実行でき、再生速度も調整できます。

## できあがるアプリ(3 ページ)

```
ホーム (/)            タスク一覧 (/tasks)        設定 (/settings)
 ├ 📋 TODO ホーム      ├ 📝 タスク一覧            ├ ⚙ 設定
 ├ [📝 タスク一覧を見る]→ ├ 入力欄 + [＋ 追加]       ├ 説明テキスト
 └ [⚙ 設定]──────┐   ├ タスク一覧表             └ [🏠 ホームへ戻る]──┐
                 │   └ [🏠 ホームへ戻る]──┐                       │
                 └──────────────┴───────── ホームへ ──┘
```

ボタンの「ページ遷移」イベントで相互に行き来でき、生成された React アプリでも
HashRouter で同じ遷移が動作します。

## シナリオ(全 8 ステップ)

1. ページを追加(タスク一覧 / 設定)
2. ホームに見出し + ナビボタン 2 つ
3. ナビボタンに遷移先(ページ遷移)を設定
4. タスク一覧ページを作成(入力 / 追加 / 一覧表 / 戻る)
5. 設定ページを作成(説明 / 戻る)
6. デザインでアクセント色を変更(全ページに即反映)
7. プレビューで画面遷移を確認(ホーム→一覧→ホーム→設定）
8. 実行モードで React を実ビルド → プレビュー

各ステップごとに `demo/screenshots/NN-*.png` を保存します。

## 前提(2 つのサーバーを起動)

```bash
# 端末1: バックエンド(http://localhost:8787)
cd backend && cargo run

# 端末2: ビルダーの dev サーバー(http://localhost:5173)
cd frontend && npm run dev
```

## 実行

```bash
cd demo
npm install        # 初回のみ(playwright + chromium を取得)
npm run demo       # ヘッド付き(ブラウザが見える)で再生
```

ウィンドウが開き、自動でビルダーを操作してアプリが組み上がります。
完成後はウィンドウを開いたまま待機します(`Ctrl+C` で終了)。

## 再生速度の調整 / オプション(環境変数)

| 変数 | 既定 | 説明 |
|---|---|---|
| **`SPEED`** | `1` | **全操作の待ち時間の倍率。`2`=ゆっくり / `0.4`=速い** |
| `HEADLESS=1` | off | ブラウザを表示せず実行(CI 向け。スクショだけ残る) |
| `SLOWMO` | `0` | Playwright ネイティブ操作の slowMo(ms) |
| `APPFORGE_API` | `http://localhost:8787` | バックエンドの URL |
| `APPFORGE_BUILDER` | `http://localhost:5173` | ビルダー dev サーバーの URL |

例:

```bash
SPEED=2 npm run demo        # プレゼン向けにゆっくり再生
SPEED=0.4 npm run demo      # サッと確認
HEADLESS=1 SPEED=0.4 npm run demo   # 画面なしでスクショだけ生成
```

画面右上のオーバーレイにも現在の速度(`速度 x◯`)が表示されます。

## しくみ(参考)

- 実行前に API で「TODO アプリ」プロジェクトを空ホームにリセット(なければ作成)。
  ビルダーは最新プロジェクトを開くので、これが対象になります。
- ビルダーへ移動後、`window.__todo` にデモ用ヘルパーとオーバーレイを注入。
- パーツ配置はビルダーと同じネイティブ DnD(`DataTransfer` + `DragEvent`)を合成して実行。
  プロパティ編集・ページ追加・イベント(ページ遷移)設定も公開 UI 経由で操作します。
- ビルダー本体には一切手を加えていません。シナリオを変えたい場合は `todo-demo.mjs` の
  `STEPS` と各ステップ本体を編集してください。
